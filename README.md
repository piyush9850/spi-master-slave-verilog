# SPI Master-Slave Controller (Verilog HDL)

## Overview

This project implements a parameterized SPI (Serial Peripheral Interface) Master-Slave communication system in Verilog HDL. The design supports all four SPI operating modes through configurable Clock Polarity (CPOL) and Clock Phase (CPHA) parameters.

The system was verified using a loopback testbench where the SPI Master and SPI Slave exchange data simultaneously over MOSI and MISO lines, demonstrating full-duplex communication.

## Features

* Parameterized support for all SPI modes:

  * Mode 0 (CPOL=0, CPHA=0)
  * Mode 1 (CPOL=0, CPHA=1)
  * Mode 2 (CPOL=1, CPHA=0)
  * Mode 3 (CPOL=1, CPHA=1)
* Full-duplex data transfer
* Configurable serial clock generation
* Master and Slave FSM implementation
* Loopback verification environment
* Synthesizable RTL design
* Testbench-based functional verification

## SPI Interface Signals

| Signal | Description                      |
| ------ | -------------------------------- |
| SCLK   | Serial Clock generated by Master |
| MOSI   | Master Out Slave In              |
| MISO   | Master In Slave Out              |
| CS     | Chip Select (Active Low)         |

## Design Architecture

### SPI Master

* Generates SCLK
* Controls CS signal
* Transmits data on MOSI
* Receives data on MISO
* Supports all CPOL/CPHA configurations

### SPI Slave

* Receives SCLK and CS from Master
* Samples incoming MOSI data
* Drives outgoing MISO data
* Supports all CPOL/CPHA configurations

### Baud Generator

* Generates SPI clock from system clock
* Configurable idle state using CPOL parameter

## Verification

Loopback testing was performed by connecting:

Master MOSI → Slave MOSI

Slave MISO → Master MISO

Master SCLK → Slave SCLK

Master CS → Slave CS

### Test Data

Master Transmit Data : 0xA5

Slave Transmit Data : 0xF1

Expected Results:

Slave Receive Data = 0xA5

Master Receive Data = 0xF1
